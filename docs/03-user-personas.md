## Overview

Developer OS serves four distinct audiences. Each persona arrives with different goals, time constraints, and success criteria. Every page on the platform exists to serve at least one of these journeys.

---

## Persona 1 — Technical Recruiter

**Primary audience.**

### Profile

| Attribute | Detail |
|-----------|--------|
| Role | Technical recruiter or talent acquisition specialist |
| Time on site | 30–60 seconds |
| Decision driver | Speed and clarity |
| Primary entry | Google search, LinkedIn, job board referral |

### Goals

- Determine if Chiedu is a fit within 30–60 seconds
- Download resume
- Verify production experience
- Find contact information

### Pain Points

- Too many resumes with no evidence of real work
- Difficult to find live projects
- Unclear technical focus

### What They Need From Developer OS

- Scannable resume with clear role alignment
- One-click resume download
- Featured project with live demo link
- Obvious contact path

### Success Criteria

- Downloads resume
- Visits Bundo
- Schedules an interview

---

## Persona 2 — Engineering Manager

### Profile

| Attribute | Detail |
|-----------|--------|
| Role | Engineering manager or tech lead evaluating candidates |
| Time on site | 3–5 minutes |
| Decision driver | Engineering maturity and team fit |
| Primary entry | Referral, portfolio link on application |

### Goals

- Evaluate engineering maturity
- Review project architecture
- Understand trade-offs and technical decisions

### Pain Points

- Portfolios that list technologies without explaining decisions
- No evidence of production deployment or maintenance
- Cannot assess how a candidate thinks under constraints

### What They Need From Developer OS

- Project pages with architecture sections
- Case studies explaining trade-offs
- GitHub links to production-quality code
- Resume confirming relevant experience

### Success Criteria

- Reads a case study
- Reviews GitHub
- Invites to a technical interview

---

## Persona 3 — Senior Engineer

### Profile

| Attribute | Detail |
|-----------|--------|
| Role | Senior or staff engineer, architect, or technical peer |
| Time on site | 5–10 minutes |
| Decision driver | Technical depth and craft |
| Primary entry | GitHub, technical blog, conference referral |

### Goals

- Assess system design ability
- Evaluate code quality and patterns
- Understand scalability decisions
- Gauge documentation and communication skills
- Identify learning mindset

### Pain Points

- Surface-level project descriptions
- No architecture diagrams or design rationale
- Missing lessons learned from real failures

### What They Need From Developer OS

- Deep case studies with architecture write-ups
- Technical blog posts (future)
- GitHub repositories with meaningful commit history
- Evidence of continuous improvement

### Success Criteria

- Completes a case study read-through
- Explores GitHub repositories
- Returns to the platform for future content

---

## Persona 4 — Founder / CTO

### Profile

| Attribute | Detail |
|-----------|--------|
| Role | Startup founder, CTO, or hiring decision-maker |
| Time on site | 2–5 minutes |
| Decision driver | Can this person ship independently? |
| Primary entry | Referral, direct outreach, portfolio link |

### Goals

- Determine if Chiedu can build independently
- Assess end-to-end product ownership
- Understand business problem-solving ability

### Pain Points

- Engineers who only code to spec
- No evidence of product thinking
- Cannot tell if someone can own a feature from idea to production

### What They Need From Developer OS

- Featured project demonstrating full ownership
- About page with engineering philosophy
- Resume showing breadth of experience
- Direct contact for conversation

### Success Criteria

- Reviews featured project and About page
- Reaches out via contact form
- Initiates hiring or collaboration conversation

|---------|----------|-----------|

| Engineering Manager | High | Case studies, architecture, GitHub |
| Senior Engineer | Medium | Deep case studies, technical content |
